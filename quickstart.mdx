<Warning>
  Only scan hosts you own or have explicit written permission to test. Unauthorized port scanning is illegal in many jurisdictions and violates most network acceptable use policies.
</Warning>

## Prerequisites

- Python **3.8** or later
- Git

No external packages are required. The scanner uses only the Python standard library.

## Get started

<Steps>
  <Step title="Clone the repository">
    Clone the repo and navigate into the project directory.

    ```bash terminal
    git clone https://github.com/your-org/python-port-scanner.git
    cd python-port-scanner
    ```

    <Note>
      No virtual environment or `pip install` is needed — the scanner has zero external dependencies.
    </Note>
  </Step>

  <Step title="Run your first scan">
    Scan ports 1–1024 on a target host. Replace `192.168.1.1` with the IP address or hostname of a machine you are authorized to scan.

    ```bash terminal
    python Port_scanner.py 192.168.1.1
    ```

    This uses the defaults: ports **1–1024**, up to **100 concurrent threads**, and a rate limit of **10 probes per second**. Results are saved automatically to a timestamped JSON file.

    To scan a narrower range faster, specify the start and end ports explicitly:

    ```bash terminal
    python Port_scanner.py 192.168.1.1 --start-port 20 --end-port 100
    ```
  </Step>

  <Step title="View the results">
    While the scan runs, each port result is printed to the terminal as it completes:

    ```text sample output
    Port 22   → ssh is open
    Port 23   → telnet is closed
    Port 25   → smtp is filtered (timeout)
    Port 53   → domain is open
    Port 80   → http is open
    Port 443  → https is open
    Port 8080 → http-alt is closed

    Port Scanning is Complete.
    Results saved to scan_results_2026-03-15_14-32-07.json
    ```

    The auto-generated JSON file contains the target host and all port results sorted by port number:

    ```json scan_results_2026-03-15_14-32-07.json
    {
        "target": "192.168.1.1",
        "scan_results": [
            {"port": 22, "status": "open"},
            {"port": 23, "status": "closed"},
            {"port": 25, "status": "filtered"},
            {"port": 53, "status": "open"},
            {"port": 80, "status": "open"},
            {"port": 443, "status": "open"},
            {"port": 8080, "status": "closed"}
        ]
    }
    ```

    <Tip>
      Pass `--output results.csv` or `--output results.txt` to save in a different format instead of JSON.
    </Tip>
  </Step>
</Steps>

## Common scan examples

<CodeGroup>

```bash Scan well-known ports
python Port_scanner.py example.com --start-port 1 --end-port 1024
```

```bash Scan a custom port range
python Port_scanner.py 10.0.0.5 --start-port 8000 --end-port 9000
```

```bash Increase thread count for speed
python Port_scanner.py 192.168.1.1 --threads 200 --start-port 1 --end-port 1024
```

```bash Save results to a CSV file
python Port_scanner.py 192.168.1.1 --output results.csv
```

```bash Save results to plain text
python Port_scanner.py 192.168.1.1 --output results.txt
```

```bash Limit scan rate (probes per second)
python Port_scanner.py 192.168.1.1 --rate-limit 5
```

</CodeGroup>

## CLI reference

| Flag | Short | Default | Description |
|------|-------|---------|-------------|
| `target` | — | *(required)* | Target IP address or hostname |
| `--start-port` | `-s` | `1` | First port in the scan range |
| `--end-port` | `-e` | `1024` | Last port in the scan range |
| `--threads` | `-t` | `100` | Maximum concurrent threads |
| `--rate-limit` | `-r` | `10` | Maximum port probes per second |
| `--output` | `-o` | auto-timestamped `.json` | Output filename (`.json`, `.txt`, or `.csv`) |

<Note>
  If you provide an `--output` filename with no extension, the scanner defaults to `.json` and appends the extension automatically.
</Note>
